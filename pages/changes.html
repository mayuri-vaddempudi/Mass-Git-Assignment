Adding Code & Git Changes

What Are Changes?

In Git, changes represent updates made to files in your repository. Each commit saves a snapshot of your project. Tracking changes helps you understand progress, collaborate with others, and maintain a clean history.

Adding code to existing repository

Cloning:

Adding code to your repository starts with cloning that repository. Cloning a repository creates a local copy that you can use to do your work. By keeping this local copy synchronized with the remote repository stored in GitHub, you're able to get the latest changes and push your updates so others can receive them.

Command: git clone repository-url

Command: cd repository-name

Create or Modify code:

Add new files or Edit existing files in your local working directory.

Check the status of your repo.This shows which files have been changed, added, or deleted.

Command: git status

Staging, Committing & Pushing changes

Staging:

Add files to the staging area (preparing them for commit).We can stage a specific file or all the changes.

Command: git add filename

Command: git add .

Commit Changes:

Commit the staged files with a meaningful message.

Command: git commit -m "Comment"

Push Changes:

Pushing meanes sending your commits to the remote repository (ex:GitHub).

Command: git push origin main (or Branch name)

Adding Code to an Existing Repository

  1. Check repository status: git status
  2. Stage changes: git add filename or git add .
  3. Commit changes: git commit -m "Commit message"
  4. Push changes to remote: git push origin branch-name
  5. Verify changes: git log --oneline

Tip: Use descriptive commit messages for clarity.
