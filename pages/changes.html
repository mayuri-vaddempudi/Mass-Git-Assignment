Git Changes & Adding Code

What Are Changes?

In Git, changes represent updates made to files in your repository. Each commit saves a snapshot of your project. Tracking changes helps you understand progress, collaborate with others, and maintain a clean history.

Adding Code to an Existing Repository

  1. Check repository status: git status
  2. Stage changes: git add filename or git add .
  3. Commit changes: git commit -m "Commit message"
  4. Push changes to remote: git push origin branch-name
  5. Verify changes: git log --oneline

Tip: Use descriptive commit messages for clarity.
